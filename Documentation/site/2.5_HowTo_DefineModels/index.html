    Defining Models

    Models.py created during startapp. Models are classes with attributes and methods as expected.

    1. Define the model in models.py
    2. Add model to settings.py
    3. Stage the model with makemigrations
    4. Push the model to db with migrate
    5. Register the model with admin site.

    Django Model Attributes & Classes (Most Common First)

    Field Types Description Example
    CharField Stores short text strings; requires max_length. name = models.CharField(max_length=100)
    TextField Stores long-form text; no length limit. description = models.TextField()
    IntegerField Stores integer values. age = models.IntegerField()
    DateTimeField Stores date and time; can auto-set on create/update. created_at = models.DateTimeField(auto_now_add=True)
    BooleanField Stores True/False. is_active = models.BooleanField(default=True)
    DecimalField Fixed-point decimal number; requires max_digits and decimal_places. price = models.DecimalField(max_digits=6, decimal_places=2)
    FloatField Stores floating-point numbers. rating = models.FloatField()
    SlugField Stores URL-friendly text; often used for slugs. slug = models.SlugField(unique=True)
    EmailField Stores email addresses; validates format. email = models.EmailField()
    URLField Stores URLs; validates format. website = models.URLField()
    FileField Stores file uploads; requires upload_to path. file = models.FileField(upload_to='uploads/')
    ImageField Stores image uploads; requires Pillow. image = models.ImageField(upload_to='images/')
    UUIDField Stores universally unique identifiers. uuid = models.UUIDField(default=uuid.uuid4, editable=False)
    Relationship Fields Description Example
    ForeignKey Many-to-one relationship; links to another model. author = models.ForeignKey(User, on_delete=models.CASCADE)
    OneToOneField One-to-one relationship between models. profile = models.OneToOneField(User, on_delete=models.CASCADE)
    ManyToManyField Many-to-many relationship between models. tags = models.ManyToManyField(Tag)
    Field Options Description Example
    unique=True Ensures field values are unique in the database. email = models.EmailField(unique=True)
    blank=True Allows the field to be left blank in forms. middle_name = models.CharField(max_length=50, blank=True)
    null=True Allows storing NULL in the database. nickname = models.CharField(max_length=50, null=True)
    default Sets a default value for a field. is_active = models.BooleanField(default=True)
    choices Enum-like choices for a field. status = models.CharField(max_length=20, choices=STATUS_CHOICES)
    help_text Adds descriptive text for forms/admin. name = models.CharField(max_length=100, help_text="Full legal name")
    verbose_name Human-readable name for a field. email = models.EmailField(verbose_name="Email Address")
    db_index=True Adds a database index for faster lookups. created_at = models.DateTimeField(db_index=True)
    primary_key=True Marks the field as the primary key. id = models.AutoField(primary_key=True)
    Model Metadata Description Example
    Meta Inner class defining model options (ordering, verbose names). class Meta: ordering = ['-created_at']

    ForeignKey(linked_model, on_delete=models.CASCADE) Database term for a reference to another record.

    Many-to-one relationship is a model/object that is related to many others. For example, an entry object must be related to a particular topic/journal.

    Defining Models

    It's a good idea to tell Django how you want it to represent an instance of a model. The str method tells Django to retrun the value assigned to the text attribute.

        class Topic(models.Model):
        """A topic the user is learning about."""
        text = models.CharField(max_length=200)
        date_added = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        """Returns a string representation of the model."""
        return self.text

    Activating Models

    You must tell Django to include our app in the overall project using settings.py.

    Include your apps before the OOB Django apps in the case that your apps need to override some default Django functionality. 

        INSTALLED_APPS = [
    # My Apps
    'learning_logs',

    # OOB Django Apps
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    ]

    Make Migrations/Migrate DB

    The makemigrations command essentially stages/commits the changes. Django figures out how it can modify the db to include the changes. 

        python manage.py makemigrations learning_logs

    Next, we migrate to the db to push our changes.

        python manage.py migrate
